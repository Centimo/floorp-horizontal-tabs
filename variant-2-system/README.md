# Floorp Horizontal Tabs — Variant 2

Кастомизация Floorp для горизонтального расположения вкладок над адресной строкой.

## Структура

- `autoconfig.js` — указатель на конфигурационный файл
- `autoconfig.cfg` — JavaScript, выполняемый при старте браузера
- `horizontal_tabs.css` — стили, инжектируемые в chrome документ

## Деплой

```bash
sudo bash ../deploy-variant-2.sh
```

Копирует файлы в `/usr/lib/floorp/` и требует перезапуска браузера.

## Архитектура

### Почему не userChrome.css

Floorp использует Shadow DOM для контейнеров вкладок. CSS из userChrome.css не может:
- Изменять атрибут `orient` (только через JS)
- Стилизовать элементы внутри Shadow DOM напрямую
- Переносить элементы в DOM (sidebar, vertical-tabs)

### autoconfig.cfg

Выполняется в привилегированном контексте при загрузке окна браузера:

1. **Читает CSS** из файла и инжектирует как `<style>`
2. **Изменяет атрибуты** `orient="horizontal"` для контейнеров вкладок и splitter
3. **Устанавливает inline стили** для свойств, которые CSS не может переопределить (Firefox внутренние стили с высоким приоритетом)
4. **Переносит элементы** в DOM: vertical-tabs в TabsToolbar, sidebar после кнопки "+"
5. **Создаёт filler** — элемент между splitter и вкладками с 3 строками для градиентов
6. **Стилизует Shadow DOM** sidebar через JavaScript
7. **MutationObserver** для:
   - Удаления "призраков" закрытых вкладок
   - Скрытия кнопки "+" при достижении лимита (18 вкладок)
   - Динамического min-width для закреплённых вкладок
   - Обновления состояния filler (hover/selected) для первого столбца

### horizontal_tabs.css

CSS переменные в `:root` для централизованной настройки цветов и размеров.

Ключевые решения:

| Проблема | Решение |
|----------|---------|
| `orient` атрибут приоритетнее CSS | JS `setAttribute()` |
| Shadow DOM изолирует стили | `::part()` селекторы для доступных частей |
| Firefox `max-height:36px` на arrowscrollbox | Inline стили через JS |
| Scroll buttons и overflow indicators | `::part()` + `display: none` |
| Двойные границы на краях строк | Убраны `border-top` у 1-й строки, `border-bottom` у 3-й |
| Левая граница первого столбца | Скрыт `overflow-start-indicator` в Shadow DOM |
| Граница между "+" и sidebar | `::before` на window-controls с `left: -1px` |
| Плавный переход на кнопке "+" | Градиент на самой кнопке, `margin-left: -5px` |
| Подсветка filler при hover/selected | 3 div-а `.filler-row` + JS отслеживание состояния |

## Параметры

| Переменная | Значение | Описание |
|------------|----------|----------|
| `--htabs-panel-height` | 120px | Высота панели вкладок |
| `--htabs-row-height` | 40px | Высота строки |
| `--htabs-pinned-cell` | 40px | Размер ячейки закреплённой вкладки |
| `--htabs-pinned-icon` | 28px | Размер иконки закреплённой вкладки |
| `--htabs-tab-width` | 150px | Ширина обычной вкладки |
| `--htabs-tab-height` | 40px | Высота обычной вкладки |
| `--htabs-selected` | rgba(100, 140, 110, 0.30) | Цвет выделенной вкладки |
| `--htabs-hover` | rgba(180, 170, 100, 0.20) | Цвет при наведении |
| `--htabs-border` | rgba(128, 128, 128, 0.3) | Цвет границ вкладок |
| `--htabs-border-accent` | rgba(128, 128, 128, 0.5) | Цвет акцентных границ |

## Лимиты

- **18 обычных вкладок** (6 колонок × 3 строки) — кнопка "+" скрывается при достижении
- **Закреплённые вкладки** — динамическое количество колонок, фиксированные 3 строки
